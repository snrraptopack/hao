# CSS as Programming Primitives — Auwla Styling System Specification

## Overview

A styling system built into the Auwla framework where CSS is authored as typed expressions,
not string declarations. Values are objects you compute with. Styles are expressions the
compiler resolves. The Vite plugin handles extraction; you just write logic.

---

## 1. Core Philosophy

Three principles:

- **CSS values are objects, not strings** — `css.px(16)` not `'16px'`. You can do arithmetic on them, pass them around, derive new values from them.
- **Styles are expressions, not declarations** — styles are written using the same variables, conditionals, and logic as the rest of your component. The compiler resolves what it can statically; the rest becomes a minimal runtime diff.
- **Everything composes** — mix, merge, extend, override. No specificity battles, no naming conventions.

---

## 2. Two Authoring Modes

### Inline — inside a render closure

Styles live directly in the component. The compiler has full context: prop types, local
variables, theme tokens. Everything in scope is available.

```tsx
function Button({ variant, disabled }) {
  let hovered = false;

  return () => (
    <button
      style={css({
        background: hovered
          ? theme.colors.primary.lighten(0.1)
          : theme.colors.primary,
        opacity: disabled ? 0.5 : 1,
        cursor: disabled ? 'not-allowed' : 'pointer',
        padding: css.compose(theme.spacing.sm, theme.spacing.md),
        borderRadius: theme.radius.sm,
        transition: css.transition({ all: { duration: css.ms(200) } })
      })}
      onMouseEnter={() => { hovered = true; }}
      onMouseLeave={() => { hovered = false; }}
    >
      {variant}
    </button>
  );
}
```

The Vite plugin walks the `css({...})` call, resolves all theme references and unit
expressions at build time, emits static CSS classes for the invariant parts, and leaves
only the genuinely dynamic branches as a small runtime object.

### External — separate style files

For shared styles, design tokens, or styles that span multiple components. No component
context; everything resolves statically. The compiler treats these as pure extraction targets.

```ts
// button.styles.ts
export const buttonStyles = css.define({
  root: {
    borderRadius: theme.radius.sm,
    fontWeight: theme.typography.weight.medium,
    border: css.border.none(),
    cursor: 'pointer'
  },
  primary: {
    background: theme.colors.primary,
    color: theme.colors.primary.contrast()
  },
  secondary: {
    background: 'transparent',
    color: theme.colors.primary,
    border: css.border({ color: theme.colors.primary, width: 1 })
  }
})
```

```tsx
// button.tsx
import { buttonStyles } from './button.styles'

return () => (
  <button style={css.merge(buttonStyles.root, buttonStyles[variant])}>
    {children}
  </button>
)
```

### Parameterized external styles

External styles that still respond to props. The compiler enumerates variants from the
type signature and emits a class for each — no runtime style computation.

```ts
// card.styles.ts
export const cardStyles = css.define((props: { elevated: boolean; size: 'sm' | 'md' | 'lg' }) => ({
  root: {
    borderRadius: theme.radius.md,
    background: theme.colors.surface,
    boxShadow: props.elevated ? theme.shadows[2] : theme.shadows[0],
    padding: css.match(props.size, {
      sm: theme.spacing.sm,
      md: theme.spacing.md,
      lg: theme.spacing.lg
    })
  }
}))
```

The compiler sees `elevated: boolean` → emits two shadow classes. Sees `size: 'sm' | 'md' | 'lg'`
→ emits three padding classes. At runtime, the framework picks the right combination from a
pre-built lookup table. Zero style computation at runtime.

## 3. Value Primitives

### Lengths & Units

```ts
const size = css.px(16)           // Length { value: 16, unit: 'px' }
const relative = css.rem(1.5)     // Length { value: 1.5, unit: 'rem' }
const viewport = css.vw(50)       // Length { value: 50, unit: 'vw' }
const zero = css.zero()

// Arithmetic — same unit returns Length
const bigger = size.add(css.px(8))       // 24px
const doubled = size.multiply(2)         // 32px
const half = size.divide(2)              // 8px

// Mixed units — returns Calc expression
const fluid = css.vw(50).subtract(css.px(100))  // calc(50vw - 100px)

// Clamping
const responsive = css.clamp(css.rem(1), css.vw(4), css.px(64))
// => clamp(1rem, 4vw, 64px)

const halfWidth = css.pct(50)   // 50%
const grid = css.fr(1)          // 1fr
```

### Colors

```ts
const blue = css.color('#3b82f6')
const teal = css.color('oklch(0.6 0.2 180)')
const semantic = theme.colors.primary

// Manipulation — all return Color, all resolve at build time
const lighter = blue.lighten(0.2)
const faded = blue.alpha(0.3)
const rotated = blue.rotate(90)

// Contrast
const textColor = blue.contrast()              // black or white, whichever passes WCAG
const safe = blue.ensureContrast(textColor, 4.5)  // adjust until AA

// Mixing
const blended = blue.mix(teal, 0.5)

// Gradients
const gradient = blue.gradient(blue.lighten(0.2), 45)
// => linear-gradient(45deg, #3b82f6, #6ba3f8)

const complex = css.gradient({
  angle: 135,
  stops: [
    [blue, 0],
    [blue.alpha(0.5), 50],
    [blue.lighten(0.3), 100]
  ]
})
```

### Typed Values

```ts
// Borders
const thin = css.border({ color: blue, width: 1, style: 'solid' })
const none = css.border.none()

// Shadows
const sm = css.shadow({ x: 0, y: 2, blur: 8, color: css.color.black.alpha(0.1) })
const inset = css.shadow.inset({ x: 0, y: 1, blur: 2 })

// Transforms — composable
const moved = css.transform({ translateX: css.px(10), scale: 1.1 })
const rotated = css.transform({ rotate: css.deg(45) })
const combined = moved.compose(rotated)

// Transitions
const smooth = css.transition({
  background: { duration: css.ms(200) },
  transform: { duration: css.ms(300), easing: css.ease('out') }
})

// Grid
const layout = css.grid({
  columns: [css.fr(1), css.px(300)],
  rows: [css.auto(), css.fr(1)],
  areas: [
    ['header', 'sidebar'],
    ['main',   'sidebar']
  ],
  gap: css.px(16)
})

// Flex
const flex = css.flex({
  direction: 'row',
  wrap: true,
  gap: theme.spacing.md,
  align: 'center',
  justify: 'space-between'
})
```

## 4. Conditionals

### Pattern matching on a value

```ts
css.match(props.variant, {
  elevated: {
    background: theme.colors.surface,
    boxShadow: theme.shadows[2]
  },
  outlined: {
    background: 'transparent',
    border: css.border({ color: theme.colors.outline, width: 1 })
  },
  ghost: {
    background: 'transparent',
    border: css.border.none()
  }
})
```

### Boolean conditions

```ts
css.when({
  [props.isSelected]: {
    background: theme.colors.selected,
    fontWeight: theme.typography.weight.bold
  },
  [props.isDragging]: {
    opacity: 0.6,
    boxShadow: theme.shadows[3]
  },
  default: {
    background: 'transparent'
  }
})
```

Both `css.match` and `css.when` are fully typed — if `variant` is `'elevated' | 'outlined' | 'ghost'`,
the compiler will error on unhandled cases if you add a new variant.

## 5. Responsive Design

### Breakpoints

```ts
css.matchBreakpoint({
  mobile: {
    padding: theme.spacing.sm,
    gridTemplateColumns: css.col(1)
  },
  tablet: {
    padding: theme.spacing.md,
    gridTemplateColumns: css.col(2)
  },
  desktop: {
    padding: theme.spacing.lg,
    gridTemplateColumns: css.col(3)
  }
})
```

### Range queries

```ts
css.matchBreakpoint({
  [css.below('tablet')]: {
    flexDirection: 'column'
  },
  [css.above('tablet')]: {
    flexDirection: 'row'
  }
})
```

### Container queries

```ts
css.matchContainer({
  narrow: { flexDirection: 'column', imageSize: css.px(40) },
  medium: { flexDirection: 'row',    imageSize: css.px(56) },
  wide:   { flexDirection: 'row',    imageSize: css.px(72) }
})
```

### User preferences

```ts
css.matchPreference({
  'prefers-color-scheme': {
    dark:  theme.colors.darkMode,
    light: theme.colors.lightMode
  },
  'prefers-reduced-motion': {
    reduce:        theme.motion.reduced,
    'no-preference': theme.motion.standard
  }
})
```

## 6. Composition

### Reusable style fragments

```ts
const withCard = css.define(({ depth = 1 }) => ({
  padding: theme.spacing.lg,
  background: theme.colors.surface,
  borderRadius: theme.radius.md,
  boxShadow: theme.shadows[depth]
}))

const withFocusRing = css.define({
  ':focus-visible': {
    outline: css.outline({ color: theme.colors.focus, width: 2, offset: 2 })
  }
})
```

### Merging

```ts
// css.merge — combines style objects, later entries win on conflict
const cardWithFocus = css.merge(
  withCard({ depth: 2 }),
  withFocusRing,
  { maxWidth: css.px(400) }
)
```

Note: `css.compose(a, b)` is an alias for `css.merge(a, b)`. Prefer `css.merge` for clarity.
`css.compose` as a shorthand for multi-value CSS properties (e.g. padding shorthand) is
handled by passing an array: `padding: [theme.spacing.sm, theme.spacing.md]`.

### Extending

```ts
const baseButton = css.define({
  borderRadius: theme.radius.sm,
  fontWeight: 500,
  border: css.border.none(),
  cursor: 'pointer'
})

const primaryButton = css.extend(baseButton, {
  background: theme.colors.primary,
  color: theme.colors.primary.contrast()
})

const largeButton = css.extend(baseButton, {
  padding: [theme.spacing.md, theme.spacing.lg],
  fontSize: theme.typography.scale.body.multiply(1.2)
})
```

## 7. Selectors & Pseudo-classes

```ts
css.define({
  list: {
    [css.children('li')]: {
      padding: [theme.spacing.sm, theme.spacing.md],
      borderBottom: css.border({ color: theme.colors.divider })
    },
    [css.child('li').first]: {
      borderTop: css.border({ color: theme.colors.divider })
    },
    [css.child('li').nth('odd')]: {
      background: theme.colors.surfaceAlt
    }
  },

  card: {
    [css.descendant('img')]: {
      borderRadius: theme.radius.sm,
      objectFit: 'cover'
    }
  },

  tooltip: {
    [css.pseudo('before')]: {
      content: css.string(''),
      position: 'absolute',
      bottom: css.pct(100),
      left: css.pct(50),
      transform: css.transform({ translateX: css.pct(-50) })
    }
  }
})
```

## 8. Animations

### Keyframe animations

```ts
css.animation({
  keyframes: [
    [0,   { opacity: 0, transform: css.transform({ scale: 0.95, translateY: css.px(10) }) }],
    [100, { opacity: 1, transform: css.transform({ scale: 1,    translateY: 0 }) }]
  ],
  duration: css.ms(300),
  easing: css.ease('out'),
  fill: 'forwards'
})
```

### Spring physics

```ts
css.spring({
  from: { opacity: 0, transform: css.transform({ scale: 0.9 }) },
  to:   { opacity: 1, transform: css.transform({ scale: 1 }) },
  stiffness: 300,
  damping: 25,
  mass: 1
})
```

### Staggered animations

```ts
css.stagger({
  children: css.selector('& > *'),
  animation: {
    keyframes: [
      [0,   { opacity: 0, transform: css.transform({ translateY: css.px(20) }) }],
      [100, { opacity: 1, transform: css.transform({ translateY: 0 }) }]
    ],
    duration: css.ms(300),
    easing: css.ease('out'),
    fill: 'backwards'
  },
  stagger: css.ms(50)
})
```

### Scroll-driven animations

```ts
css.scrollTimeline({
  axis: 'block',
  animate: {
    opacity:   css.tween(0, 1),
    transform: css.tween(
      css.transform({ scale: 0.8, translateY: css.px(40) }),
      css.transform({ scale: 1,   translateY: 0 })
    )
  }
})
```

## 10. How the Compiler Handles It

The Vite plugin (`auwla/vite`) processes all `css(...)` and `css.define(...)` calls at build time.

### What resolves at build time

- All theme token references — `theme.colors.primary` → concrete hex
- All unit arithmetic — `css.px(8).multiply(2)` → `16px`
- All color operations — `.lighten()`, `.alpha()`, `.contrast()` → concrete values
- All static style objects in `css.define({...})`
- All variant branches in parameterized `css.define((props: {...}) => ...)` — the compiler
  enumerates every combination from the prop type union and emits a class for each

### What stays at runtime

- Boolean switches between pre-emitted classes (`hovered ? classA : classB`)
- Truly dynamic values that cannot be enumerated (user-input dimensions, JS-computed sizes)
  — these become CSS custom properties with a minimal JS setter

### What the developer never sees

- Class name generation or collision
- Vendor prefixes
- Specificity management
- Dead CSS — only styles reachable from the component tree are emitted

## 11. API Consistency Reference

| Operation | API |
|---|---|
| Merge style objects | `css.merge(a, b, c)` |
| Extend a base style | `css.extend(base, overrides)` |
| Multi-value shorthand (padding etc.) | `padding: [v, h]` or `[top, right, bottom, left]` |
| Branch on a union prop | `css.match(value, cases)` |
| Branch on boolean conditions | `css.when({ [condition]: style, default: style })` |
| Branch on breakpoint | `css.matchBreakpoint(cases)` |
| Branch on container size | `css.matchContainer(cases)` |
| Branch on user preference | `css.matchPreference(cases)` |
| Define reusable fragment | `css.define(styleObject)` |
| Define parameterized fragment | `css.define((props: T) => styleObject)` |
| Inline styles in component | `css({ ... })` |
